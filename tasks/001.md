# ЗАДАЧА

Разработать систему для **обнаружения пропущенных научных цитат (missing citation detection)**.

На вход подаётся новая научная статья (PDF или текст + список литературы).
На выходе система должна вернуть список **релевантных статей из базы, которые логично должны быть процитированы, но отсутствуют в списке литературы**.

База научных статей в которой мы ищем формируется из имеющихся pdf файлов которые хотим разобрать в md формат.


---

# ОБЩАЯ АРХИТЕКТУРА

Pipeline:

1. Ingestion (индексация базы статей)
2. Claim Extraction (из новой статьи)
3. Retrieval (поиск кандидатов)
4. Filtering (удаление уже процитированных)
5. Re-ranking (cross-encoder)
6. LLM Judge (финальная оценка)
7. Explanation (почему нужно цитировать)

---

# ШАГ 1 — INGESTION

## Вход:

* PDF или структурированный JSON статьи

## Что сделать:

1. Извлечь текст

2. Выделить:

   * title
   * abstract
   * sections (если возможно)
   * references (список литературы)

3. Разбить на chunks:

   * размер: 200–400 токенов
   * overlap: 50 токенов

4. Для каждого chunk:

   * посчитать embedding
   * сохранить в Qdrant

## Payload в Qdrant:

{
id: uuid,
vector: embedding,
payload: {
paper_id: str,
title: str,
chunk_text: str,
section: str,
year: int,
doi: str (если есть)
}
}

---

# ШАГ 2 — CLAIM EXTRACTION

Реализовать через LLM.

## Вход:

текст статьи (лучше sections: intro + related work + methods)

## Выход:

список claims:

[
{
"text": "...",
"type": "method | background | result"
}
]

## Prompt:

"Extract atomic scientific claims from the text.
Each claim must be a statement that could require a citation."

Ограничить:

* 10–20 claims максимум

---

# ШАГ 3 — RETRIEVAL

Для каждого claim:

1. Получить embedding
2. Найти top-100 в Qdrant
3. Объединить результаты (deduplicate по paper_id)

---

# ШАГ 4 — FILTERING

Удалить:

* статьи, уже присутствующие в references
* exact match по DOI / названию (fuzzy matching)

---

# ШАГ 5 — RERANK (CROSS-ENCODER)

Для каждой пары:
(claim, candidate_paper)

Посчитать score релевантности.

Оставить top-20 кандидатов.

---

# ШАГ 6 — LLM JUDGE

Для каждого кандидата:

## Prompt:

"You are a scientific reviewer.

Given:

1. A claim from a paper
2. A candidate paper

Decide if the candidate paper SHOULD be cited.

Criteria:

* Is it foundational?
* Does it introduce the method?
* Does it provide evidence for the claim?

Answer in JSON:
{
"should_cite": true/false,
"confidence": 0-1,
"reason": "..."
}
"

Отобрать:

* только should_cite = true
* confidence > 0.6

---

# ШАГ 7 — EXPLANATION

Сгенерировать финальный ответ:

[
{
"paper_title": "...",
"reason": "...",
"related_claim": "...",
"confidence": ...
}
]
